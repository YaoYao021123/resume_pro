# 首次设置向导

本向导帮助你在 **15-30 分钟内**完成初始设置，之后每次生成简历只需粘贴 JD 即可。

---

## 第一步：填写个人基本信息

打开 `data/profile.md`，按照说明填写以下内容：

- **基本信息**：姓名、邮箱、电话
- **教育背景**：学校、专业、GPA、时间段、主修课程
- **技能**：编程语言、工具、外语水平
- **获奖情况**：竞赛、奖学金等

> **提示**：课程列表尽量填多，生成简历时会根据岗位自动筛选 4-5 门最相关的展示。

---

## 第二步：建立经历库

每段实习/工作经历单独建一个文件，存放在 `data/experiences/` 目录下。

**文件命名规则：** `{序号}_{公司名}.md`，例如：
```
data/experiences/
├── 01_Google.md
├── 02_麦肯锡.md
└── 03_字节跳动.md
```

复制 `data/experiences/_template.md` 作为起点，按说明填写每段经历的：
- 基本信息（公司、职位、时间、部门）
- 标签（便于与 JD 关键词匹配）
- 工作内容（建议用 STAR 格式，多写，生成时会按 JD 筛选）

> **核心原则 — 多存少取**：每段经历尽量记录所有工作内容（4-8 条），
> 生成简历时系统只会选出最匹配 JD 的 2-3 条展示。
> 原始文件永远不会被修改，方便下次重新筛选。

---

## 第三步（可选）：上传原始工作文件

如果你有原始工作材料（分析报告、项目文档、会议纪要等），可以放入：

```
data/work_materials/
└── {公司名}/
    ├── 报告.pdf
    └── 分析.md
```

系统生成简历时会优先参考这些原始材料（而不是你自己总结的描述），提取更真实、更有说服力的业绩数据。

---

## 第四步：验证设置

在 Claude Code 中运行以下命令，让系统检查你的数据是否完整：

```
/resume --check
```

系统会提示哪些字段未填写或格式有问题。

---

## 第五步：生成第一份简历

找一份目标岗位的 JD，粘贴给 Claude Code：

```
帮我针对这个岗位生成简历：

[粘贴JD内容]
```

---

## 常见问题

**Q: 经历内容写多详细？**
A: 越详细越好。每条工作内容都用 STAR 格式写（Situation → Task → Action → Result），包含具体数字。系统会根据 JD 自动筛选最匹配的 2-3 条。

**Q: 没有量化数据怎么办？**
A: 尽量描述规模（文件数量、服务客户数、涉及金额等）。没有的不要捏造，系统只会改写和筛选你提供的真实内容。

**Q: 可以同时投多家公司吗？**
A: 可以。每次生成的简历都独立存放在 `output/` 目录下，互不影响。

**Q: 生成的 LaTeX 看起来不对怎么办？**
A: 特殊字符需要转义：`&` 写成 `\&`，`%` 写成 `\%`。如果报错，检查 `output/{目录}/resume.tex` 中是否有未转义的字符。

## 多人档案管理

系统支持管理多个人的简历数据（比如帮朋友/家人做简历）。

### 方式一：Web UI 管理

启动 `python3 web/server.py`，侧边栏顶部的"当前档案"下拉菜单可以切换人员。点击齿轮按钮打开档案管理弹窗，可添加/删除人员。

### 方式二：命令行

```bash
# 迁移旧数据（首次自动运行，也可手动）
python3 tools/migrate_to_multi_person.py

# 为指定人员生成简历
python3 tools/generate_resume.py --person alice '你的JD文本'
```

### 数据结构

多人模式下，每个人的数据独立存放：

```
data/
├── persons.json              # 人员注册表
├── _shared/experiences/      # 共享模板
├── default/                  # 默认人员（迁移自旧数据）
│   ├── profile.md
│   ├── experiences/
│   └── work_materials/
└── alice/                    # 其他人员
    ├── profile.md
    ├── experiences/
    └── work_materials/
```

输出也按人员隔离：`output/{person_id}/{公司}_{岗位}_{日期}/`
