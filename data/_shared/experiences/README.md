# 经历库说明

## 文件组织规则

每段实习 / 工作 / 项目经历放一个 `.md` 文件，按时间倒序编号：

```
experiences/
├── _template.md        ← 模板（不要删，复制使用）
├── 01_最近公司.md      ← 最新经历
├── 02_第二段经历.md
└── 03_更早的经历.md
```

## 填写原则

| 原则 | 说明 |
|------|------|
| **多存** | 每段经历尽量记录所有工作内容，不限条数 |
| **少取** | 生成简历时只选 2-3 条最匹配 JD 的展示 |
| **不删** | 原始文件永远不修改，每次生成可重新筛选 |
| **量化** | 每条内容尽量包含具体数字（数量、比例、金额、时间） |

## 标签填写参考

标签用于将你的经历与 JD 关键词做匹配，分几类填写效果最好：

- **行业**：互联网、金融、医疗、教育、新能源、消费品、咨询……
- **职能**：产品、运营、数据分析、研究、销售、市场、技术开发……
- **工具/技术**：Python、SQL、Excel、Figma、Tableau、Bloomberg……
- **业务关键词**：用户增长、商业化、B端、C端、AI、出海……
